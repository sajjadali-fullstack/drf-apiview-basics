DRF Views
============

1. APIView ==========> similar to FBV [Function Base View]
2. ViewSet ==========> similarto CBV [Class Base View]


APIVew is provide more customization 
ViewSet is provide less customization and Ready made things
RECOMMENDED is ---> ViewSet when u want to create a Rapid development


Q. What is APIView?
>>> It is a most basic class to build REST APIs
>>> It is the child class of django view class 
>>> We have to write complete code of business logic and hence programmer having complete control on the logic.
>>> We can understand flow of execution very clearly.
>>> Best suitable for complex operations like working with multiple data source, callling other APIs etc.
>>> We have to define url mapping manually.
>>> It allows us to use standard HTTP methods as function like get(), post(), put(), etc.

Q. What is ViewSet?

>>> With the help of this methods we can create APIs very easily 

>>> list() ------------> get all records
>>> retrive() -----------> to get single record
>>> create() ---------> to get a new record
>>> update() ---------> to update a record 
>>> partial_update() ---------> to update few fields of data.
>>> destroy() ------> to delete a record.

=> Developing very simple APIs
